KMUToken 배포

Remix IDE에서 KMUToken 컨트랙트 선택 후 배포

// Remix IDE 설정 Compiler: 0.8.20+commit Environment: Injected Provider - MetaMask Network: Sepolia Testnet // 배포 후 콘솔에서 복사 KMUToken address: 0xYOUR_TOKEN_ADDRESS
1,000,000 KMU 자동 발행 faucet() 으로 추가 수령 가능
2

StakingContract 배포

배포 시 KMUToken 주소를 생성자 인자로 입력

// Deploy 탭 → Deploy 버튼 옆 입력란에: _stakingToken: "0xYOUR_TOKEN_ADDRESS" // 배포 완료 후 복사 StakingContract address: 0xYOUR_STAKING_ADDRESS
3

approve() 호출

KMUToken 컨트랙트에서 스테이킹 컨트랙트에 사용 허가

// KMUToken 컨트랙트 → approve 함수 spender: "0xYOUR_STAKING_ADDRESS" amount: "100000000000000000000" // 100 KMU (18 decimals) // approve 완료 → Etherscan에서 트랜잭션 확인

approve 없이 stake() 호출 시 ERC20: insufficient allowance 에러 발생

4

stake() / unstake() 호출

스테이킹 컨트랙트에서 실행 — 제출용 트랜잭션 해시 획득

stake(amount)

amount: "100000000000000000000" // 100 KMU

unstake(amount)

amount: "50000000000000000000" // 50 KMU

컨트랙트 주요 함수

stake(uint256) 토큰 예치 — approve 선행 필수
unstake(uint256) 예치 토큰 일부/전체 출금
claimReward() 누적 보상 토큰 수령
emergencyWithdraw() 보상 포기 후 원금 즉시 회수
getStakeInfo(address) 예치량 / 미수령 보상 / 총 수령 조회
